Implementing Retrieval-Augmented Generation (RAG) with Knowledge Graph

Tools: Python, OpenAI (free tier), LangChain, Neo4j

In my previous article, I constructed a knowledge graph with the help of LLMs. This knowledge graph now becames an external knowledge source for the LLM. We will be using a knowledge graph and embeddings in a vector database to augment LLM responses. We can use the either a knowledge graph or vector embeddings alone to conduct RAG, or use a combination of both.

I am on the free tier on the OpenAI platform, which is subjected to rate limits on API usage. I am also restricted from using the gpt-4 model, so I will be using gpt-3.5.
